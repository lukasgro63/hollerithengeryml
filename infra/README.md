# infra

Operational assets for HollerithEnergyML: the Docker Compose stack, the
Caddy reverse proxy configuration, and the GitOps pipeline that deploys
via Portainer.

## Files

| File | Purpose |
| --- | --- |
| `docker-compose.yml` | Local development stack (web + api on a shared bridge network). |
| `docker-compose.prod.yml` | Production stack consumed by Portainer as a Git-backed stack. |
| `Caddyfile` | Reverse proxy for `/api/*` → api, `/*` → web. HTTP only. |

## Deployment flow (GitOps)

```
git push main
   │
   ▼
GitHub Actions CI                (ruff · pytest · tsc · next build)
   │
   ▼
GitHub Actions build-push        (buildx → ghcr.io/lukasgro63/hollerithengeryml/{web,api}:latest)
   │
   ▼
POST $PORTAINER_WEBHOOK_URL      (no SSH involved)
   │
   ▼
Portainer pulls docker-compose.prod.yml from the repo
Portainer pulls new images from ghcr.io
Portainer restarts the stack
   │
   ▼
smoke test http://$SITE_DOMAIN/api/v1/health
```

Portainer is configured with the stack as a Git repository reference, so it
always reconciles to the state in this directory on `main`.

## Required GitHub configuration

- Secret: `PORTAINER_WEBHOOK_URL` — the webhook generated by Portainer when the
  stack is created with GitOps updates enabled.
- Variable: `SITE_DOMAIN` — the production hostname or IP (e.g. `46.224.134.32`).

## Local development

```bash
docker compose -f infra/docker-compose.yml up --build
```

Web is reachable at `http://localhost:3000` and api at `http://localhost:8000`.

## Production topology

```
internet
   │
   ▼
Caddy :80    (reverse proxy, security headers, gzip/zstd)
   │
   ├── /api/*  →  api  :8000  (FastAPI, read_only, tmpfs /tmp)
   │
   └── /*      →  web  :3000  (Next.js standalone)
```

All three services share the `hollerith` Docker network. Caddy is the only
container that publishes a port on the host (`80`).
